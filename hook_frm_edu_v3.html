What this app does

This app illustrates the hook-length formula for multi-symbol shortest paths. Given a word P (e.g. 22100) and its reverse-sorted version E (e.g. 00122), the formula computes:

  • d(P,E) — the shortest distance (number of adjacent swaps needed to go from E to P)
  • The number of shortest paths from E to P using only adjacent symbol swaps

The Young diagram displays the shape determined by your symbol counts. Each cell shows its hook length; the formula uses the product of hook lengths and factorials.

Sliders

  • r — number of symbol types (2–6)
  • a0, a1, a2, … — count of each symbol in the word
  • Cell — cell size (px) in the Young diagram
  • Top, Canvas W/H, Paths W/H, Result W — layout dimensions (config, canvas, paths panel, result panel)

Checkboxes

  • Show hook lengths — show hook length in each cell
  • Show formula steps — show step-by-step computation (n!, hook product, etc.)
  • Step-by-step — expandable explanation of the theorem
  • Quiz mode — practice by answering questions about d(P,E) and path count
  • Show paths — list all shortest paths from E to P in the right panel
  • Show subgraph — draw the path graph (layered nodes) under the Young diagram

"Show paths" / "Show subgraph" — both rely on enumerating all shortest paths. That drives the main cost: memory and CPU are proportional to f^λ.
